1. `compute_subject_waic(input_data_arr: np.ndarray, observed_arr: np.ndarray, choices_arr: np.ndarray, idata: az.InferenceData, model_name: str) -> float`
   - This is the critical function that works around the pm.Potential WAIC blocker.
   - Build the logp Op using `build_logp_ops_2level` or `build_logp_ops_3level` (based on model_name)
   - Extract posterior samples from `idata.posterior`:
     - For 2-level: omega_2, beta, zeta (get beta, not log_beta)
     - For 3-level: omega_2, omega_3, kappa, beta, zeta
   - Each is shape (n_chains, n_draws)
   - **FAST PATH:** Try to use the JAX JIT-compiled logp function directly rather than the PyTensor Op to avoid per-call overhead. The `_jax_logp` closure is internal to the factory, so rebuild the Op to get access to `_jit_logp`. Actually, the Op's `perform` method calls `_jit_logp` — so call `logp_op(pt.as_tensor_variable(o2), ...).eval()` for each posterior sample.
   - **Vectorization strategy:** Iterate over (chain, draw) pairs. For each, extract scalar parameter values and evaluate the Op. Collect all logp values into array shape (chains, draws).
   - Create xarray DataArray: `xr.DataArray(loglike_vals[:, :, np.newaxis], dims=["chain", "draw", "loglike_dim_0"])`
   - Add to idata: `idata.add_groups({"log_likelihood": {"loglike": ll_da}})`
   - Call `az.waic(idata, var_name="loglike")`
   - Return `waic_result.elpd_waic` (scalar, higher = better model)
   - Add progress logging: print every 100 evaluations for tracking
   - Note: For 180 participants x 4 chains x 1000 draws = 720,000 evaluations total; each is fast (JAX JIT-compiled) but budget ~10-30 min for full batch

2. `compute_batch_waic(sim_df: pd.DataFrame, fit_results: dict[str, pd.DataFrame], idata_dict: dict[str, dict[tuple, az.InferenceData]], model_names: list[str] = None) -> pd.DataFrame`
   - Takes the full batch data and pre-loaded InferenceData objects
   - For each (participant_id, group, session) x model_name: call compute_subject_waic
   - Returns DataFrame with columns: participant_id, group, session, model, elpd_waic
   - model_names defaults to ["hgf_2level", "hgf_3level"]
   - Note: idata_dict is keyed as `idata_dict[model_name][(participant_id, group, session)]`
   - **IMPORTANT:** This function reconstructs input_data_arr, observed_arr, and choices_arr
     from sim_df for each participant before calling compute_subject_waic. It filters sim_df
     by (participant_id, group, session) and extracts the trial-level arrays needed by the
     logp Op. This is the correct entry point for the pipeline script since .nc idata files
     do not store these arrays.
   - **Alternative simpler approach:** Accept a function signature that takes (sim_df, model_name) and rebuilds the Op + evaluates logp for each posterior sample from stored idata. If idata was not stored, the pipeline script must re-run fitting with idata storage. Document this constraint clearly.

**Simpler alternative for compute_subject_waic if Op evaluation is too slow:**

Instead of evaluating the full PyTensor Op per sample, rebuild just the raw JAX function. The `build_logp_ops_*` factories create a `_jax_logp` closure and then wrap it in Op classes. We can replicate the JAX function construction outside the Op wrapper:

```python
# In bms.py, create a helper that builds the raw JAX logp function
# without wrapping in PyTensor Op. Then use jax.vmap over posterior samples.
```

This approach is significantly faster but requires duplicating the network construction logic. Choose the simpler Op-based approach first; optimize only if runtime exceeds 1 hour for full batch.

Follow project conventions: `from __future__ import annotations`, NumPy docstrings,
type hints, line length 88.

3. `run_group_bms(elpd_matrix: np.ndarray, model_names: list[str], group_label: str = "all") -> dict`
   - elpd_matrix shape: (n_subjects, n_models) — each entry is elpd_waic for that subject under that model
   - Validate shape: `assert elpd_matrix.ndim == 2 and elpd_matrix.shape[1] == len(model_names)`
   - Call `GroupBMC(elpd_matrix).get_result()`
   - **Runtime discovery:** Print `dir(result)` and `result.__dict__` on first call to discover exact attribute names. The expected fields from MATLAB reference are: alpha, exp_r, xp, pxp, bor. The Python package may use different names.
   - Return a dict with keys: `alpha` (Dirichlet posterior params), `exp_r` (expected model frequencies), `xp` (exceedance probabilities), `pxp` (protected exceedance probabilities), `bor` (Bayesian Omnibus Risk), `model_names`, `group_label`, `n_subjects`
   - If groupBMC was not installable and VB was implemented from scratch, wrap that instead
   - If pxp is not available from the package, compute it: `pxp = (1 - bor) * xp + bor / n_models`

4. `run_stratified_bms(waic_df: pd.DataFrame, model_names: list[str]) -> dict[str, dict]`
   - Run BMS for full sample and for each unique group in waic_df
   - Build elpd_matrix by pivoting waic_df: for each subject, one column per model
   - Full sample: `run_group_bms(elpd_matrix, model_names, group_label="all")`
   - Per group: filter waic_df by group, build matrix, run BMS
   - Return dict keyed by group_label ("all", "healthy_control", "post_concussion")
   - Log warning for per-group runs: "Per-group BMS with N={n} subjects has limited power (exploratory)"

5. `plot_exceedance_probabilities(bms_results: dict[str, dict], save_path: Path | None = None) -> plt.Figure`
   - Multi-panel figure: one subplot per group label (all, healthy_control, post_concussion)
   - Each subplot: grouped bar chart with EP and protected EP for each model
   - Horizontal dashed line at 1/n_models (chance level)
   - Title includes group label and n_subjects
   - Annotate BOR value as text below bars
   - Save to save_path if provided (PNG, 150 dpi)
   - Use `matplotlib.use("Agg")` at module top
