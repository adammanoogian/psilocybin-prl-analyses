Create `src/prl_hgf/analysis/recovery.py` with three public functions:

1. `build_recovery_df(sim_df: pd.DataFrame, fit_df: pd.DataFrame, exclude_flagged: bool = True, min_n: int = 30) -> pd.DataFrame`
   - Reduce sim_df to one row per (participant_id, group, session) using
     `groupby(["participant_id", "group", "session"])[true_* cols].first().reset_index()`
   - The true_* columns are: `true_omega_2`, `true_omega_3`, `true_kappa`, `true_beta`, `true_zeta`
   - If `exclude_flagged=True`, filter fit_df to rows where `flagged == False` before pivoting
   - Pivot fit_df to wide form: `pivot_table(index=["participant_id", "group", "session"], columns="parameter", values="mean")`
   - After pivot, columns will be named by parameter (omega_2, omega_3, kappa, beta, zeta)
   - Merge true_params with fitted_wide on ["participant_id", "group", "session"]
   - Return merged DataFrame; also log how many participants were excluded if `exclude_flagged=True`
   - Raise ValueError if merge produces zero rows (empty intersection)
   - **N >= min_n guard (REC-04):** After merge, check `len(recovery_df) >= min_n`. If below
     threshold, raise `ValueError(f"Recovery requires at least {min_n} participants, got
     {len(recovery_df)}. Increase simulation batch size or reduce exclusions.")`. Default
     min_n=30 ensures adequate statistical power for correlation-based recovery metrics.
     Pass min_n=0 to disable the guard (useful for unit tests with small synthetic data).

2. `compute_recovery_metrics(recovery_df: pd.DataFrame, param_map: dict[str, str] | None = None) -> pd.DataFrame`
   - Default `param_map = {"omega_2": "true_omega_2", "omega_3": "true_omega_3", "kappa": "true_kappa", "beta": "true_beta", "zeta": "true_zeta"}`
   - For each (fitted_col, true_col) pair present in recovery_df:
     - Mask NaN/Inf values from both arrays
     - Compute Pearson r and p-value via `scipy.stats.pearsonr`
     - Compute bias = `np.mean(recovered - true)` (signed mean error)
     - Compute RMSE = `np.sqrt(np.mean((recovered - true)**2))`
   - Return DataFrame with columns: `parameter`, `r`, `p`, `bias`, `rmse`, `n`, `passes_threshold`
   - `passes_threshold` is True if `abs(r) >= 0.7` (from config analysis.recovery_r_threshold)
   - Skip parameters not present in recovery_df (2-level model won't have omega_3, kappa)

3. `compute_correlation_matrix(recovery_df: pd.DataFrame, param_cols: list[str] | None = None) -> pd.DataFrame`
   - Default param_cols = whichever of ["omega_2", "omega_3", "kappa", "beta", "zeta"] exist in recovery_df
   - Return `recovery_df[param_cols].corr(method="pearson")`
   - This is the posterior-mean cross-parameter correlation matrix

Update `__init__.py` to import from recovery module. Follow project conventions:

Python 3.10+ syntax, line length 88.
