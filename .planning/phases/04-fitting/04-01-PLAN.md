**ops.py — Custom PyTensor Ops for both model variants**

Create `src/prl_hgf/fitting/ops.py` implementing the two-Op split pattern
from 04-RESEARCH.md. Two public factory functions:

1. `build_logp_ops_2level(input_data_arr, observed_arr, choices_arr)` returns
   `(LogpOp, n_trials)`. The Op accepts `(omega_2, beta, zeta)` as PyTensor
   variables and returns scalar logp.

2. `build_logp_ops_3level(input_data_arr, observed_arr, choices_arr)` returns
   `(LogpOp, n_trials)`. The Op accepts `(omega_2, omega_3, kappa, beta, zeta)`.

Implementation details for EACH factory function:
- Build a fresh Network via build_2level_network() or build_3level_network()
- Call net.input_data(input_data=input_data_arr, observed=observed_arr) ONCE
  to trigger scan_fn creation
- Freeze base_attributes = net.attributes, scan_fn = net.scan_fn
- Build scan_inputs tuple: values = tuple of (n_trials, 1) arrays split from
  input_data_arr columns; observed_cols = tuple of observed_arr columns as int;
  time_steps = np.ones(n_trials); fourth element = None
- Define inner `_jax_logp(omega_2, beta, zeta, ...)` function that:
  - Shallow-copies base_attributes (dict(base_attrs))
  - For 2-level: injects omega_2 into nodes 1, 3, 5 tonic_volatility
  - For 3-level: additionally injects omega_3 into node 6 tonic_volatility,
    kappa into node 6 volatility_coupling_children as jnp.array([kappa]*3),
    AND kappa into nodes 1,3,5 volatility_coupling_parents as jnp.array([kappa])
  - Runs lax.scan(scan_fn, attrs, scan_inputs) to get node_trajectories
  - Extracts expected_mean from nodes 0, 2, 4 (binary input nodes, NOT belief
    nodes) and stacks to (n_trials, 3) as mu1
  - Computes softmax-stickiness log-likelihood:
    prev = jnp.concatenate([jnp.array([-1]), choices_jax[:-1]])
    stick = (prev[:, None] == jnp.arange(3)[None, :]).astype(jnp.float32)
    logits = beta * mu1 + zeta * stick
    lp = jax.nn.log_softmax(logits, axis=1)
    return jnp.sum(lp[jnp.arange(n_trials), choices_jax])
  - NaN guard: wrap return with jnp.where(jnp.isnan(result), -jnp.inf, result)
    (return -inf not inf, since this is logp not surprise)
- JIT-compile: _jit_val_grad = jax.jit(jax.value_and_grad(_jax_logp, argnums=...))
  and _jit_logp = jax.jit(_jax_logp)
- Define inner _GradOp(Op) class with make_node and perform methods
- Define inner _LogpOp(Op) class with make_node, perform, and grad methods
  where grad delegates to _GradOp instance
- Return (_LogpOp(), n_trials)

CRITICAL anti-patterns to avoid:
- Never deepcopy attributes (breaks JAX traceability)
- Never use expected_mean from continuous-state nodes (1,3,5) — use binary
  input nodes (0,2,4) for mu1 (sigmoid-transformed probability in [0,1])
- omega_2 >= 0 causes NaN — this is handled by the prior, but the Op should
  return -inf (via NaN guard) if it happens anyway
- For 3-level kappa injection: MUST inject into BOTH volatility_coupling_children
  on node 6 AND volatility_coupling_parents on nodes 1, 3, 5

**models.py — PyMC model factory functions**

Create `src/prl_hgf/fitting/models.py` with two factory functions:

1. `build_pymc_model_2level(input_data_arr, observed_arr, choices_arr)`:
   - Calls build_logp_ops_2level to get the Op
   - Creates pm.Model with:
     - omega_2 = pm.TruncatedNormal("omega_2", mu=-3.0, sigma=2.0, upper=0.0)
     - log_beta = pm.Normal("log_beta", mu=0.0, sigma=1.5)
     - beta = pm.Deterministic("beta", pm.math.exp(log_beta))
     - zeta = pm.Normal("zeta", mu=0.0, sigma=2.0)
     - pm.Potential("loglike", logp_op(omega_2, beta, zeta))
   - Returns (model, var_names) where var_names = ["omega_2", "beta", "zeta"]

2. `build_pymc_model_3level(input_data_arr, observed_arr, choices_arr)`:
   - Same pattern but adds:
     - omega_3 = pm.TruncatedNormal("omega_3", mu=-6.0, sigma=2.0, upper=0.0)
     - kappa = pm.TruncatedNormal("kappa", mu=1.0, sigma=0.5, lower=0.01, upper=2.0)
     - pm.Potential("loglike", logp_op(omega_2, omega_3, kappa, beta, zeta))
   - Returns (model, var_names) where var_names = ["omega_2", "omega_3", "kappa", "beta", "zeta"]

Both functions should suppress the g++ warning at module level:
