Create `src/prl_hgf/simulation/` package with `__init__.py` and `agent.py`.

**`agent.py`** must contain two public functions:

1. **`sample_participant_params(group_cfg, session_cfg, session_idx, rng)`**
   - Draws individual parameters from `GroupConfig` distributions: `rng.normal(group_cfg.omega_2.mean, group_cfg.omega_2.sd)` for each of omega_2, omega_3, kappa, beta, zeta.
   - If `session_idx > 0` (non-baseline), adds session deltas from `SessionConfig`: `session_cfg.omega_2_deltas[session_idx - 1]` etc. Note: SessionConfig only has deltas for omega_2, kappa, beta, zeta (no omega_3 deltas in config).
   - Clips ALL parameters to model bounds AFTER adding deltas (order: sample -> add delta -> clip). Bounds from prl_analysis.yaml: omega_2 [-8.0, 2.0], omega_3 [-12.0, 0.0], kappa [0.01, 2.0], beta [0.01, 20.0], zeta [-5.0, 5.0].
   - Returns `dict[str, float]` with keys "omega_2", "omega_3", "kappa", "beta", "zeta".
   - Type signature: `(group_cfg: GroupConfig, session_cfg: SessionConfig, session_idx: int, rng: np.random.Generator) -> dict[str, float]`

2. **`simulate_agent(net, trials, beta, zeta, rng)`**
   - Trial-by-trial simulation loop using the attribute carry pattern from RESEARCH.md.
   - At each trial:
     a. Read prior beliefs from `net.attributes[node]["expected_mean"]` for INPUT_NODES (0, 2, 4) BEFORE calling input_data.
     b. Compute softmax with stickiness using numpy (not JAX — simulation path doesn't need gradients): `logits = beta * p_reward + zeta * stick`, `probs = exp(logits - max) / sum(exp(logits - max))`.
     c. Sample choice via `rng.choice(3, p=probs)`.
     d. Generate reward via `generate_reward(choice, trial.cue_probs, rng)` from `prl_hgf.env.simulator`.
     e. Build 1-trial input arrays: `inp_t = np.zeros((1, 3))`, `obs_t = np.zeros((1, 3), dtype=int)`, set `inp_t[0, choice] = float(reward)`, `obs_t[0, choice] = 1`.
     f. Call `net.input_data(input_data=inp_t, observed=obs_t)`.
     g. CRITICAL: carry state forward with `net.attributes = net.last_attributes`.
     h. Update prev_choice.
   - First trial: prev_choice sentinel is -1 (stickiness term = 0 for all cues).
   - Returns a `SimulationResult` dataclass (frozen) with fields: `choices: list[int]`, `rewards: list[int]`, `beliefs: list[tuple[float, float, float]]` (the prior p_reward read at each trial, for verification — optional but useful for tests).
   - Type signature: `(net: Network, trials: list[Trial], beta: float, zeta: float, rng: np.random.Generator) -> SimulationResult`

**`__init__.py`** exports: `simulate_agent`, `sample_participant_params`, `SimulationResult`.

**Imports:** Use `from prl_hgf.models.hgf_2level import INPUT_NODES` for node indices. Use `from prl_hgf.env.simulator import Trial, generate_reward`. Use `from prl_hgf.env.task_config import GroupConfig, SessionConfig`. Use numpy (not JAX) for all simulation math.

**Conventions:** `from __future__ import annotations` at top. NumPy-style docstrings. Type hints with Python 3.10+ syntax. Line length 88. Run `ruff check --fix` and `ruff format` after writing.

**Parameter sampling tests:**

1. `test_sample_params_baseline_keys_and_types` — Call `sample_participant_params` with session_idx=0, verify returned dict has exactly the 5 expected keys, all values are floats.

2. `test_sample_params_within_bounds` — Sample 100 parameter sets with a fixed seed. Assert ALL values are within the clipping bounds defined in PARAM_BOUNDS. Use both post_concussion and healthy_control group configs from `load_config()`.

3. `test_sample_params_session_deltas_applied` — With a deterministic seed that produces a known baseline omega_2 value, verify that session_idx=1 produces omega_2 shifted by exactly the post_dose delta (within float tolerance). Compare: `params_s0["omega_2"] + delta == params_s1["omega_2"]` (if not clipped).

4. `test_sample_params_reproducible` — Same seed produces identical params dict.

5. `test_sample_params_clip_after_delta` — Construct a scenario where the baseline sample is near a bound and the delta pushes it further toward the bound. Verify the result is clipped (not the unclipped value).

**Simulation tests (these require pyhgf + JAX, take ~5-8s each):**

6. `test_simulate_agent_output_shape` — Run `simulate_agent` for 1 session (420 trials) with default healthy_control params. Verify: len(choices)==420, len(rewards)==420, all choices in {0,1,2}, all rewards in {0,1}.

7. `test_simulate_agent_reproducible` — Same network params + same seed + same trials produce identical choices and rewards.

8. `test_simulate_agent_high_beta_accuracy` — Build a 3-level network with healthy_control omega_2=-3.0, omega_3=-6.0, kappa=1.0. Simulate with beta=5.0, zeta=0.0. Compute accuracy (choice==best_cue) for the LAST HALF of each stable (acquisition) phase across all sets. Assert mean accuracy > 0.80. This is the key success criterion from the roadmap. Use `trial.phase_label == "stable" and trial.phase_name.startswith("acquisition")` to select stable acquisition trials, then take the last half of each phase.

9. `test_simulate_agent_reversal_accuracy_drop` — Same setup as above. Compute accuracy for volatile (reversal) phases. Assert volatile accuracy < stable late-phase accuracy (reversals cause temporary drops).

10. `test_simulate_agent_first_trial_no_stickiness` — Run 1 trial. Verify that with prev_choice=-1 sentinel, the first trial's choice probabilities are proportional to `exp(beta * p_reward)` only (no stickiness term). This can be checked by running the same trial twice with different zeta values — the choice should be identical when there's only 1 trial (no prev_choice to stick to).
