**Modify fit_batch in batch.py:**

Add an optional parameter `return_idata: bool = False` to `fit_batch()`. When True:
- Collect InferenceData objects in a dict keyed by `(participant_id, group, session)` tuple
- Return type becomes `tuple[pd.DataFrame, dict[tuple, az.InferenceData]]` when return_idata=True
- For backward compatibility: when return_idata=False (default), return only the DataFrame as before
- Implementation: after `fit_participant` returns `(idata, summary_rows, flagged)`, if return_idata, store `idata_dict[(participant_id, group, session)] = idata`
- For failed fits: store None in idata_dict for that key
- Update the docstring to document the new parameter and overloaded return type

**Add RESULTS_DIR to config.py** if not already present:
- `RESULTS_DIR = PROJECT_ROOT / "results"`
- `VALIDATION_DIR = RESULTS_DIR / "validation"`

**Create scripts/05_run_validation.py:**

This is the Phase 5 pipeline entry point. Structure:

```python
"""Phase 5: Parameter recovery validation and Bayesian model comparison.

Loads simulation data and fitting results, computes recovery metrics and
scatter plots, runs random-effects BMS (Rigoux et al. 2014), and saves
all outputs to results/validation/.

Usage:
    conda run -n ds_env python scripts/05_run_validation.py
    conda run -n ds_env python scripts/05_run_validation.py --skip-waic
"""
```

The script should:

1. **Parse CLI args**: `--skip-waic` flag to skip the slow WAIC computation (useful for
   testing recovery-only). `--sim-path` and `--fit-path` to point to custom CSV files
   (defaults to `results/simulated_data.csv` and `results/fit_results_{model}.csv`).

2. **Load data**: Read sim_df CSV, read fit_df CSVs for each model (hgf_2level, hgf_3level).
   If CSV files don't exist, print instructions to run scripts 03 and 04 first.

3. **Recovery analysis** (for each model separately):
   - `build_recovery_df(sim_df, fit_df, min_n=30)` with exclude_flagged=True — the min_n=30
     default enforces REC-04. If ValueError is raised due to insufficient N, log the error
     and exit with a clear message: "Need at least 30 non-flagged participants. Got {N}.
     Re-run simulation with larger batch size."
   - `compute_recovery_metrics(recovery_df)`
   - `compute_correlation_matrix(recovery_df)`
   - Print metrics table to console
   - Print warning if any |corr| > 0.8
   - Print explicit omega_3 / kappa recovery caveat
   - Save metrics CSV to `results/validation/recovery_metrics_{model}.csv`

4. **Recovery plots** (for each model):
   - `plot_recovery_scatter(...)` saved to `results/validation/recovery_scatter_{model}.png`
   - `plot_correlation_matrix(...)` saved to `results/validation/correlation_matrix_{model}.png`

5. **WAIC computation** (unless --skip-waic):
   - **CRITICAL:** Do NOT call compute_subject_waic directly by loading .nc idata files.
     The .nc files do not contain input_data_arr, observed_arr, or choices_arr which are
     required by compute_subject_waic.
   - **Correct approach:** Use `compute_batch_waic(sim_df, fit_results, idata_dict, model_names)`
     which accepts sim_df and reconstructs the required arrays internally for each participant.
   - Load sim_df (already loaded in step 2).
   - Load idata objects: check if `results/idata/{model}/{participant_id}_{group}_{session}.nc`
     files exist. If not, re-run fit_batch with `return_idata=True` on the existing sim_df
     to obtain InferenceData objects. Store them for subsequent use.
   - Pass sim_df + idata_dict to `compute_batch_waic()` which handles array reconstruction
     and calls compute_subject_waic per participant.
   - Save WAIC results to `results/validation/waic_results.csv`

6. **BMS** (if WAIC results available):
   - `run_stratified_bms(waic_df, model_names=["hgf_2level", "hgf_3level"])`
   - Print BMS results table (model, exp_r, xp, pxp, bor) for each group
   - `plot_exceedance_probabilities(bms_results)` saved to `results/validation/bms_exceedance.png`
   - Save BMS summary to `results/validation/bms_summary.csv`

7. **Summary**: Print final summary to console:
   - Recovery: which parameters pass r > 0.7 threshold
   - BMS: winning model per group
   - Any concerns (high correlations, poor omega_3 recovery)

Use config.py paths (RESULTS_DIR, VALIDATION_DIR). Create output directories with
`mkdir(parents=True, exist_ok=True)`.

Follow the existing script pattern from `04_fit_participants.py` for CLI structure,
logging, and main() function.
