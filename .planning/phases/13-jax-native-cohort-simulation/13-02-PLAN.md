**Part A: Add `simulate_cohort_jax` to `jax_session.py`**

Add to `__all__`: `"simulate_cohort_jax"`.

Add function:
```python
def simulate_cohort_jax(
    params_batch: dict[str, jnp.ndarray],
    cue_probs_arr: jnp.ndarray,
    rng_keys_batch: jnp.ndarray,
) -> tuple[jnp.ndarray, jnp.ndarray, jnp.ndarray]:
```

- `params_batch`: dict with keys `"omega_2"`, `"omega_3"`, `"kappa"`, `"beta"`, `"zeta"`,
  each shape `(P,)` — one value per participant
- `cue_probs_arr`: shape `(n_trials, 3)` — SHARED across all participants (same trial sequence)
- `rng_keys_batch`: shape `(P, 2)` — one distinct key per participant
- Returns: `(choices, rewards, diverged)` where choices/rewards have shape `(P, n_trials)`,
  diverged has shape `(P,)`

Implementation — uses the factory pattern established in Plan 01. `_build_session_scanner`
builds the Network once outside vmap; `_run_session` is the pure-JAX kernel that gets
vmapped. No conditional restructure needed — Plan 01 already split into this structure.

```python
def simulate_cohort_jax(params_batch, cue_probs_arr, rng_keys_batch):
    """vmap across participants using factory pattern from Plan 01."""
    scan_fn, base_attrs = _build_session_scanner()
    _vmapped = jax.vmap(
        lambda o2, o3, k, b, z, rk: _run_session(
            scan_fn, base_attrs, o2, o3, k, b, z, cue_probs_arr, rk
        ),
        in_axes=(0, 0, 0, 0, 0, 0),
    )
    return _vmapped(
        params_batch["omega_2"], params_batch["omega_3"],
        params_batch["kappa"], params_batch["beta"],
        params_batch["zeta"], rng_keys_batch,
    )
```

Note: `cue_probs_arr` is captured in the lambda closure (shared across all participants).
The vmap axes are: omega_2(0), omega_3(0), kappa(0), beta(0), zeta(0), rng_key(0).
`scan_fn` and `base_attrs` are also in the closure (built once by factory).

**Part B: Update `batch.py` to use JAX path (Option B: full vmap)**

Replace the inner loop in `simulate_batch` with the JAX cohort path. The phase goal is
"full cohort in one compiled kernel," so we commit to Option B: stack per-participant
`cue_probs_arr` and vmap over them.

**Why Option B:** Each participant-session has a different `env_seed` producing a different
trial sequence (different `cue_probs_arr`). Option A would loop per-session in Python,
defeating the phase goal. Option B stacks per-participant `cue_probs_arr` into shape
`(P, n_trials, 3)` and vmaps over it alongside params and keys.

**Variable-length sessions:** If sessions ever have different `n_trials` per participant,
vmap cannot handle ragged arrays. Document that variable-length sessions fall back to
per-session `simulate_session_jax` calls. For now, all sessions have the same `n_trials`
(config-driven), so this is not an issue.

**Implementation:**

1. Keep the upfront seed derivation (`rng_master`, `all_seeds`) — these are used for
   parameter sampling via NumPy (sample_participant_params stays NumPy-based).
2. Remove `_prewarm_jit()` function and its call.
3. **First pass — collect all params and trial sequences:** Loop over (group, participant,
   session). For each, call `sample_participant_params` (NumPy), `generate_session` (to get
   trial sequence), and accumulate into batch arrays:
   - `all_omega_2`, `all_omega_3`, `all_kappa`, `all_beta`, `all_zeta` — Python lists of floats
   - `all_cue_probs` — Python list of `[t.cue_probs for t in trials]` (each is `(n_trials, 3)`)
   - `all_rng_keys` — Python list of `jax.random.PRNGKey(sim_seed)`
   - `all_trials` — Python list of trial lists (for DataFrame assembly)
   - `all_metadata` — Python list of (group_name, participant_id, session_name, params_dict)
4. **Stack into batch arrays:**
   ```python
   params_batch = {
       "omega_2": jnp.array(all_omega_2),
       "omega_3": jnp.array(all_omega_3),
       "kappa": jnp.array(all_kappa),
       "beta": jnp.array(all_beta),
       "zeta": jnp.array(all_zeta),
   }
   cue_probs_batch = jnp.stack(all_cue_probs)  # shape (P_total, n_trials, 3)
   rng_keys_batch = jnp.stack(all_rng_keys)    # shape (P_total, 2)
   ```
5. **Single vmapped call:**
   ```python

   _vmapped = jax.vmap(
       lambda o2, o3, k, b, z, cp, rk: _run_session(
           scan_fn, base_attrs, o2, o3, k, b, z, cp, rk
       ),
       in_axes=(0, 0, 0, 0, 0, 0, 0),
   )
   all_choices, all_rewards, all_diverged = _vmapped(
       params_batch["omega_2"], params_batch["omega_3"],
       params_batch["kappa"], params_batch["beta"],
       params_batch["zeta"], cue_probs_batch, rng_keys_batch,
   )
   ```
   Note: unlike `simulate_cohort_jax` (which shares cue_probs via `None` axis),
   here `cue_probs_batch` is per-participant so `in_axes=0` for all args.
