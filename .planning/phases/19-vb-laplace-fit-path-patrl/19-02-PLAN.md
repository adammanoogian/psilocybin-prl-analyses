<objective>
Build the InferenceData factory that Phase 19's Laplace fit pipeline will
use to package `(mode_params, covariance)` into an `az.InferenceData` with
the same posterior schema as the NUTS path so Plan 18-05's exporters
(`export_subject_trajectories`, `export_subject_parameters`) consume it
without modification.

This module is independent of the MAP optimizer and Hessian computation
(Plan 19-03 consumes it) — it is pure packaging, testable in isolation
against a hand-constructed `(mode, cov)` pair.

Key architectural choice (from research Q5 + OQ1): emit dim name
`"participant_id"` NATIVELY. The NUTS path's `_samples_to_idata` emits
`"participant"` (line 1606 in `hierarchical.py`) while the consumer
`export_subject_trajectories` reads `"participant_id"` (line 181, 345 in
`export_trajectories.py`). Phase 19 does NOT fix `_samples_to_idata`
(parallel-stack invariant bans that); instead the Laplace path ships the
correct consumer-facing dim name natively.

Purpose: Unblock Plan 19-03 by providing a fully-tested packaging surface
with a clean input contract. All downstream InferenceData quirks (coords,
dims, deterministic beta, sample_stats skeleton) are resolved here once.

Output:
- `src/prl_hgf/fitting/laplace_idata.py` with `build_idata_from_laplace`
  public function and module-private helpers.
- `tests/test_laplace_idata.py` with 6-8 tests covering shape parity,
  dim-name parity with the export consumer, deterministic beta, HDI
  computation, sample_stats skeleton presence, and var-name contracts
  for 2-level and 3-level models.

> "Package a Laplace approximation `(mode, Σ)` into an
> `az.InferenceData` with the same posterior schema as the NUTS path.
>

> kappa, mu3_0), then adds `beta = exp(log_beta)` as a deterministic
> transform.
>
> Emits dim name `'participant_id'` natively — matches what
> `src/prl_hgf/analysis/export_trajectories.py` reads. The NUTS path's
> `_samples_to_idata` uses the older dim name `'participant'`; this
> module's choice sidesteps that latent producer/consumer mismatch (OQ1
> in Phase 19 research, to be hotfixed in a separate Phase 18 follow-up)."

Public signature:
```python
def build_idata_from_laplace(
    mode: dict[str, np.ndarray],          # var -> shape (P,) native-space mode
    cov: np.ndarray,                      # (P*K, P*K) covariance in flatten order
    param_names: tuple[str, ...],         # canonical order, one of the two above
    participant_ids: list[str],           # len P
    n_pseudo_draws: int = 1000,
    rng_key: int = 0,
    diagnostics: dict[str, float] | None = None,
) -> az.InferenceData:
    """Build an ArviZ InferenceData from a Laplace approximation.

    Parameters
    ----------
    mode : dict[str, np.ndarray]
        Posterior mode, one (P,) array per variable. Keys must match
        ``param_names`` exactly.
    cov : np.ndarray
        Flat covariance of shape ``(P*K, P*K)`` where P is the number of
        participants and K is ``len(param_names)``. Ordering matches
        ``jax.flatten_util.ravel_pytree`` of a dict keyed by
        ``param_names`` in insertion order.
    param_names : tuple[str, ...]
        Canonical parameter order. Must be ``_PARAM_ORDER_2LEVEL`` or
        ``_PARAM_ORDER_3LEVEL``.
    participant_ids : list[str]
        Participant identifiers, length P.
    n_pseudo_draws : int, default 1000
        Number of pseudo-samples to draw from ``N(mode, cov)``. Does NOT
        affect the Laplace approximation quality (the Gaussian is exact);
        only affects Monte-Carlo noise in sample-based summaries like HDI.
    rng_key : int, default 0
        Seed for the numpy Generator used to draw the pseudo-samples.
    diagnostics : dict[str, float] | None, default None
        Optional diagnostic summary (converged, n_iterations,
        logp_at_mode, hessian_min_eigval, ridge_added). If provided,
        stashed in the ``sample_stats`` group as scalars broadcast over
        ``(chain=1, draw=K)``.

    Returns
    -------
    az.InferenceData
        Posterior with vars ``omega_2, log_beta, beta`` (2-level) or
        ``omega_2, log_beta, beta, omega_3, kappa, mu3_0`` (3-level),
        each dim ``(chain=1, draw=n_pseudo_draws,
        participant_id=P)``. ``sample_stats`` group present (populated
        from ``diagnostics`` or empty skeleton).
    """
```

Implementation steps:

1. Validate `param_names` is one of the two canonical tuples; raise
   `ValueError` with expected vs actual if not.
2. Validate `mode` keys match `param_names` exactly (same set, same
   order); raise `ValueError` with expected vs actual if not.
3. Let `P = len(participant_ids)`; `K = len(param_names)`.
4. Validate `cov.shape == (P * K, P * K)`; raise with expected vs actual.
4a. **Validate per-variable mode shapes (m4 guard).** Immediately after
    step 2, iterate mode values:
    ```python
    for k, v in mode.items():
        v_arr = np.asarray(v)
        if v_arr.ndim != 1 or v_arr.shape[0] != P:
            raise ValueError(
                f"mode[{k!r}] must be shape ({P},); got {v_arr.shape}"
            )
    ```
    Catches the common bug of passing a `(1, P)` or `(P, 1)` array from
    a caller. Tested in Task 2.
5. Build flat mode: `mode_flat = np.concatenate([mode[v] for v in
   param_names])` is WRONG — the `ravel_pytree` order interleaves by
   participant. Instead, match the ravel_pytree convention: for a dict
   `{v1: (P,), v2: (P,), ...}`, ravel_pytree concatenates
   `v1_flat, v2_flat, ...` where each `vI_flat` is `v_I` itself. So
   `mode_flat = np.concatenate([np.asarray(mode[v]) for v in
   param_names])` is CORRECT and matches `ravel_pytree` on a dict keyed
   in insertion order with value shapes `(P,)`.
   Document this explicitly in a comment with a one-line unit-test check
   in Task 2.
6. Draw `n_pseudo_draws` samples from `N(mode_flat, cov)` using
   `rng = np.random.default_rng(rng_key); samples = rng.multivariate_normal(
   mode_flat, cov, size=n_pseudo_draws)`. Shape `(n_pseudo_draws, P*K)`.
7. Unflatten: reshape per variable. The `ravel_pytree` layout means
   samples columns `[i*P : (i+1)*P]` correspond to variable
   `param_names[i]` across participants. So for each `i, var in
   enumerate(param_names)`:
   `posterior[var] = samples[:, i*P:(i+1)*P][None, :, :]` giving shape
   `(1, n_pseudo_draws, P)`.
8. Add deterministic `beta = exp(log_beta)` if `"log_beta"` is in
   `param_names`:
   `posterior["beta"] = np.exp(posterior["log_beta"])`.
9. Build coord + dim dicts:
   ```python
   coords = {
       "chain": [0],
       "draw": np.arange(n_pseudo_draws),
       "participant_id": list(participant_ids),
   }
   dims = {var: ["participant_id"] for var in posterior}
   ```
10. Build `sample_stats` dict. If `diagnostics` is provided, broadcast
    each scalar to shape `(1, n_pseudo_draws)`:
    ```python
    sample_stats = {
        k: np.full((1, n_pseudo_draws), float(v))
        for k, v in diagnostics.items()
    }
    ```
    If `diagnostics is None`, pass an empty dict but still include the
    `sample_stats` group by passing a minimal key `{"laplace": np.ones((1,
    n_pseudo_draws), dtype=bool)}` (a marker that this is Laplace output
    so consumers can introspect).
11. Return:
    ```python
    idata = az.from_dict(
        posterior=posterior,
        sample_stats=sample_stats,
        coords=coords,
        dims=dims,
    )
    return idata
    ```
