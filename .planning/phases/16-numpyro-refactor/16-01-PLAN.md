  **ADD `build_logp_fn_batched(model_name, n_trials)` as a NEW function (place after `build_logp_ops_batched`):**

  This factory:
  1. Builds the pyhgf Network once to capture `base_attrs` and `scan_fn` (same as in `build_logp_ops_batched`)
  2. Defines `_single_logp_3level` and `_single_logp_2level` using the same pure JAX logic as the existing versions. The `base_attrs`, `scan_fn`, and `n_trials` are the ONLY closures (static model structure, not data)
  3. Creates `_batched_logp = jax.vmap(_single_participant_logp, in_axes=tuple([0] * (n_params + 4)))` (same vmap pattern)
  4. Returns a function `batched_logp_fn(*params, input_data, observed, choices, trial_mask)` that calls `_batched_logp` and sums. CRITICAL: This function takes data as ARGUMENTS, not from closure. This is the key difference from `build_logp_ops_batched`.

  Returned function signature for 3-level:
  `batched_logp_fn(omega_2, omega_3, kappa, beta, zeta, input_data, observed, choices, trial_mask) -> scalar`

  For 2-level:
  `batched_logp_fn(omega_2, beta, zeta, input_data, observed, choices, trial_mask) -> scalar`

  **ADD two numpyro model functions (module-level, after `build_logp_fn_batched`):**

  `_numpyro_model_3level(input_data, observed, choices, trial_mask, n_participants, batched_logp_fn)`:
  - Priors (matching EXACTLY the current PyMC model):
    - `omega_2 = numpyro.sample("omega_2", dist.TruncatedNormal(loc=-3.0, scale=2.0, high=0.0).expand([n_participants]))`
      NOTE: The current PyMC model uses `pm.TruncatedNormal("omega_2", mu=-3.0, sigma=2.0, upper=0.0)` which means lower=-inf. For numpyro TruncatedNormal, `low` defaults to -inf so just omit it.
    - `omega_3 = numpyro.sample("omega_3", dist.TruncatedNormal(loc=-6.0, scale=2.0, high=0.0).expand([n_participants]))`
    - `kappa = numpyro.sample("kappa", dist.TruncatedNormal(loc=1.0, scale=0.5, low=0.01, high=2.0).expand([n_participants]))`
    - `log_beta = numpyro.sample("log_beta", dist.Normal(0.0, 1.5).expand([n_participants]))`
    - `beta = numpyro.deterministic("beta", jnp.exp(log_beta))`
    - `zeta = numpyro.sample("zeta", dist.Normal(0.0, 2.0).expand([n_participants]))`
  - Custom logp: `logp = batched_logp_fn(omega_2, omega_3, kappa, beta, zeta, input_data, observed, choices, trial_mask)`
  - `numpyro.factor("hgf_loglike", logp)`

  `_numpyro_model_2level(input_data, observed, choices, trial_mask, n_participants, batched_logp_fn)`:
  - Same pattern but only omega_2, log_beta/beta, zeta priors
  - `logp = batched_logp_fn(omega_2, beta, zeta, input_data, observed, choices, trial_mask)`
  - `numpyro.factor("hgf_loglike", logp)`

  **ADD numpyro imports (deferred):**
  Keep deferred imports for numpyro/arviz inside `fit_batch_hierarchical()` to match the existing deferred-import pattern for heavy dependencies. Import `numpyro`, `numpyro.distributions as dist`, `from numpyro.infer import MCMC, NUTS` inside the function or at the top of the numpyro model functions if they are called only from `fit_batch_hierarchical`.

  **REWRITE `fit_batch_hierarchical()` to use the numpyro path:**
  - Build the logp function via `build_logp_fn_batched(model_name, n_trials)`
  - Select the numpyro model function based on `model_name`
  - Create `kernel = NUTS(model_fn, ...)`
  - Create `mcmc = MCMC(kernel, num_warmup=n_tune, num_samples=n_draws, num_chains=n_chains)`
  - Call `mcmc.run(rng_key, input_data=..., observed=..., choices=..., trial_mask=..., n_participants=..., batched_logp_fn=batched_logp_fn)`
  - Convert via `az.from_numpyro(mcmc, dims=..., coords=...)` to produce InferenceData with `participant` dim and `participant_group`/`participant_session` coords (same structure as before)
  - `chain_method="vectorized"` for 4 chains on single GPU
  - `sampler` parameter kept but raises `DeprecationWarning` if `sampler="pymc"` is passed; numpyro path is always used
