Create `scripts/11_write_recommendation.py` with `from __future__ import annotations`, the usual `sys.path.insert(0, ...)`, and argparse CLI.

**CLI arguments:**
- `--input-dir` (default: `RESULTS_DIR / "power"`) — directory containing `power_master.csv`, `power_a_summary.csv`, `power_b_summary.csv`
- `--precheck-dir` (default: `RESULTS_DIR / "power"`) — directory containing `trial_sweep_results.csv` and `power_eligible_params.csv`
- `--output-dir` (default: `RESULTS_DIR / "power"`) — where to write `recommendation.md`
- `--bf-threshold` (default: 6.0, type float) — BF threshold for power reporting (same as the plotting script)
- `--power-target` (default: 0.80, type float) — target power level for recommendation

**Define a `generate_recommendation()` function** that takes the DataFrames and parameters as arguments and returns a string (the Markdown content). This makes it testable without file I/O.

**`generate_recommendation(master_df, power_a_df, power_b_df, precheck_sweep_df, eligibility_df, bf_threshold, power_target) -> str`:**

The returned Markdown string should have this structure:

```markdown
# Power Analysis Recommendation

## Study Design Summary

- **Task**: PRL pick_best_cue (3 cues, 4 phases)
- **Groups**: Psilocybin vs Placebo (both post-concussion syndrome)
- **Sessions**: Baseline, Post-dose, Follow-up
- **Primary model**: 3-level binary HGF
- **Primary parameter**: omega_2 (tonic volatility)
- **Primary contrast**: Difference-in-differences (post_dose - baseline)

## Recommended Sample Size

**N = {recommended_N} per group** ({recommended_N * 2} total)

At d = 0.5 (medium effect), this provides:
- P(BF > {bf_threshold}) = {p_value:.0%} for the primary contrast (did_postdose)
- P(correct model selection via BMS) = {bms_value:.0%}

### Finding the Recommendation

The recommended N is the smallest grid value where P(BF > {bf_threshold}) >= {power_target:.0%} for d = 0.5 on the primary contrast. {If no grid point crosses, state: "No tested N achieves {power_target:.0%} power at d = 0.5; the largest tested N = {max_N} achieves {max_p:.0%}. Consider increasing N or accepting a larger effect size target."}

## Recommended Trial Count

**{trial_count} trials per session**

Based on the trial count sweep (Phase 9 prechecks), this is the minimum trial count where all power-eligible parameters (omega_2, beta, kappa, zeta) achieve recovery r >= 0.7.

{If trial_sweep_results.csv is not found, state: "Trial count recommendation pending — precheck sweep results not found."}

## Eligible Parameters

Parameters included in the power analysis based on recovery eligibility (r >= 0.7 threshold):

| Parameter | Recovery r | Status |
|-----------|-----------|--------|
{For each row in eligibility_df: | {parameter} | {r:.2f} | {"PASS" if eligible else "FAIL"} |}

{If eligibility_df is None, state: "Eligibility data not available — precheck results not found."}

## Power Summary Table

| N/group | d = 0.3 | d = 0.5 | d = 0.7 |
|---------|---------|---------|---------|
{For each N in grid: | N | P(BF>{bf_threshold}) | ... | ... |}

## BMS Discriminability

| N/group | P(correct model) |
|---------|-------------------|
{For each N: | N | P(BMS correct) |}

## Exclusion Rate

- Total iterations in master dataset: {n_total}
- Mean exclusion rate (R-hat > 1.05 or mean_rhat > 1.05): {rate:.1%}
- {If rate > 5%: "WARNING: Exclusion rate exceeds 5% — consider increasing MCMC budget."}
- {If rate <= 5%: "Exclusion rate is within acceptable bounds (< 5%)."}

## Caveats

### omega_3 (Meta-Volatility)

omega_3 power estimates are labeled **exploratory -- upper bound**. Parameter recovery for omega_3 with binary PRL data yields r ~ 0.67 in the literature, below the r >= 0.7 threshold. Naive BFDA power estimates for omega_3 would be inflated by 20-40 percentage points. Any group effects on omega_3 should be interpreted cautiously and treated as hypothesis-generating.

### Effect Size Assumptions

Power estimates assume a Cohen's d = 0.5 (medium) effect of psilocybin on omega_2 changes from baseline to post-dose. This is a placeholder informed by prior psychedelic research. The actual effect size is unknown.

### MCMC Settings

Power sweep used reduced MCMC settings (2 chains, 500 draws, 500 tune) for computational feasibility. These are adequate for BF estimation but inference-grade analyses should use the full settings (4 chains, 2000 draws, 2000 tune).

## Evidence Generated

- Power Analysis A: {n_a_cells} (N x d) cells, {n_iterations} iterations each
- Power Analysis B: {n_b_levels} N levels with BMS comparison
- Precheck: {n_precheck_points} trial count grid points evaluated
- Total MCMC fits: ~{total_fits} (estimated from grid size x 2 models)
```

**Logic for finding recommended N:**
1. Recompute P(BF > bf_threshold) from `master_df` raw `bf_value` column for `sweep_type == "did_postdose"` (same approach as the plotting script)
2. Filter to `effect_size == 0.5` (or closest if 0.5 not in grid)
3. Sort by `n_per_group` ascending
4. Find first N where `p_bf_exceeds >= power_target`
5. If none found, use the largest N tested and note it

**Logic for trial count:**
1. Read `trial_sweep_results.csv` if it exists
2. For each trial_count, check if all non-omega_3 parameters have r >= 0.7
3. Return the minimum such trial_count
4. If file not found, return None and note in report

**Logic for eligible parameters section:**
1. If `eligibility_df` is not None, iterate rows (columns: parameter, r, eligible)
2. For each row, format a table row with parameter name, r value (2 decimal places), and "PASS"/"FAIL" based on the `eligible` boolean column
3. If `eligibility_df` is None, emit placeholder text stating eligibility data not available

**Logic for exclusion rate:**
1. From `master_df`, compute fraction of rows where `mean_rhat > 1.05`
2. Report as percentage

**`main()` function:**
1. Parse args
2. Read CSVs from input_dir and precheck_dir
3. Call `generate_recommendation(...)` to get the Markdown string
4. Write to `output_dir / "recommendation.md"`
5. Print the recommendation to stdout as well
