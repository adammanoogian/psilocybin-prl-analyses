<objective>
Rewrite the core fitting pipeline in hierarchical.py to use BlackJAX NUTS instead
of NumPyro MCMC.  Add three new private functions (_build_log_posterior,
_run_blackjax_nuts, _samples_to_idata) and rewrite fit_batch_hierarchical to
orchestrate them.  Add blackjax dependency.

Purpose: Eliminate the ~1800s per-call JIT recompilation caused by NumPyro's
internal function-object recreation.  BlackJAX compiles the NUTS step function
once via jax.jit, then reuses it across all MCMC steps and power-sweep
iterations.

Output: A working fit_batch_hierarchical that uses BlackJAX by default, with
numpyro as a fallback sampler.  Pure JAX log-posterior.  Multi-GPU pmap support.
</objective>

2. **Add `_build_log_posterior` to hierarchical.py** (after the existing
   `build_logp_fn_batched` function, before the NumPyro model functions):

   This function takes the `batched_logp_fn` returned by `build_logp_fn_batched`,
   the data arrays (input_data, observed, choices, trial_mask), n_participants,
   and model_name.  It returns a `logdensity_fn(params_dict) -> scalar` suitable
   for BlackJAX.

   Implementation details:
   - Use `numpyro.distributions.TruncatedNormal` and `numpyro.distributions.Normal`
     for prior log_prob computation (standalone, no model context).  These are
     pure JAX and match the existing priors in `_numpyro_model_3level/2level`
     EXACTLY.  Note: numpyro.distributions is used standalone for prior log-probs
     only — there is no NumPyro MCMC dependency in this path.
   - Prior specs for 3-level: omega_2 ~ TruncatedNormal(loc=-3.0, scale=2.0, high=0.0),
     omega_3 ~ TruncatedNormal(loc=-6.0, scale=2.0, high=0.0),
     kappa ~ TruncatedNormal(loc=1.0, scale=0.5, low=0.01, high=2.0),
     log_beta ~ Normal(0.0, 1.5), zeta ~ Normal(0.0, 2.0).
   - Prior specs for 2-level: omega_2 ~ TruncatedNormal(loc=-3.0, scale=2.0, high=0.0),
     log_beta ~ Normal(0.0, 1.5), zeta ~ Normal(0.0, 2.0).
   - The returned `logdensity_fn` accepts a single dict `params` with keys
     matching the parameter names (omega_2, omega_3, kappa, log_beta, zeta for
     3-level; omega_2, log_beta, zeta for 2-level).  Each value has shape `(P,)`.
   - Inside logdensity_fn: compute `beta = jnp.exp(params["log_beta"])`, then
     call `batched_logp_fn(omega_2, omega_3, kappa, beta, zeta, input_data,
     observed, choices, trial_mask)` for 3-level (analogous for 2-level).
   - Data arrays are captured in the closure (they have fixed shape per call).
   - Return `prior_lp + likelihood_lp` as a scalar.

   Signature:
   ```python
   def _build_log_posterior(
       batched_logp_fn,
       input_data: jnp.ndarray,
       observed: jnp.ndarray,
       choices: jnp.ndarray,
       trial_mask: jnp.ndarray,
       n_participants: int,
       model_name: str = "hgf_3level",
   ) -> callable:
   ```

3. **Add `_run_blackjax_nuts` to hierarchical.py** (after `_build_log_posterior`):

   This function runs BlackJAX NUTS with window_adaptation warmup and lax.scan
   sampling.  It handles multi-GPU via pmap or single-GPU via vmap.

   Implementation details:
   - Import `blackjax` at module level (deferred import inside function body to
     avoid import errors when blackjax not installed -- use the same pattern as
     numpyro imports in this file: import inside the function).
   - Warmup: Use `blackjax.window_adaptation(blackjax.nuts, logdensity_fn,
     target_acceptance_rate=target_accept, is_mass_matrix_diagonal=True)`.
   - **Warmup strategy (single warmup):** Run warmup ONCE with the first
     chain's key.  Replicate the adapted state across chains with different
     RNG keys for sampling.  Average the adapted step_size and
     inverse_mass_matrix across chains is NOT needed since we run a single
     warmup — use the single warmup's adapted parameters directly.
     Build `nuts = blackjax.nuts(logdensity_fn, step_size, inverse_mass_matrix)`.
   - Multi-chain strategy: Check `jax.device_count()`.  If >= n_chains, use
     pmap (one chain per device).  Otherwise, use vmap on single device.
   - Sampling loop: `lax.scan` with `@jax.jit` decorated `one_step` function.
     The one_step must return `(state, (state, info))` to collect NUTSInfo
     (Research Pitfall 7).
   - For vmap path: vmap the one_step across chains.  Initial states come from
     the single warmup's adapted state, replicated with different RNG keys.
   - For pmap path: pmap the per-chain sampling function.  Initial states
     distributed across devices.  Use `jax.pmap` on a function that takes
     (rng_key, initial_state) and runs the lax.scan loop.
   - Collect `states.position` (dict of arrays) and `infos.is_divergent` +
     `infos.acceptance_rate`.

   Signature:
   ```python
   def _run_blackjax_nuts(
       logdensity_fn,
       initial_position: dict[str, jnp.ndarray],
       rng_key: jnp.ndarray,
       n_tune: int = 1000,
       n_draws: int = 1000,
       n_chains: int = 4,
       target_accept: float = 0.95,
   ) -> tuple:  # (positions_dict, sample_stats_dict, n_chains_actual)
   ```

   Return shape convention:
   - positions_dict: dict of arrays shaped (n_chains, n_draws, P)
   - sample_stats_dict: dict with "diverging" (n_chains, n_draws) bool and
     "acceptance_rate" (n_chains, n_draws) float
   - n_chains_actual: int (may differ from n_chains if pmap path changes it)
