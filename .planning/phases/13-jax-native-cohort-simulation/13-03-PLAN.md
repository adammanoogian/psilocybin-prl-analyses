**Module docstring:**
```
"""VALID-04: Statistical equivalence between legacy and JAX simulation paths.

Verifies that ``simulate_agent`` (NumPy for-loop) and ``simulate_session_jax``
(JAX ``lax.scan``) produce statistically equivalent choice frequency
distributions per cue per phase over 100 replicates.

Per-trial exact match is NOT expected: the two paths use different RNG streams
(NumPy PCG64 vs JAX ThreeFry) and different floating-point accumulation orders.
Only aggregate statistics are compared via two-sample KS tests.

Runtime: ~5-10 minutes (100 replicates x 420 trials each).
"""
```

**Imports:**
- `from __future__ import annotations`
- `import numpy as np`
- `import jax`
- `import jax.numpy as jnp`
- `import pytest`
- `from collections import defaultdict`
- `from scipy.stats import ks_2samp`
- `from prl_hgf.env.task_config import load_config`
- `from prl_hgf.env.simulator import generate_session`
- `from prl_hgf.models.hgf_3level import build_3level_network`
- `from prl_hgf.simulation.agent import simulate_agent`
- `from prl_hgf.simulation.jax_session import simulate_session_jax`

**Constants:**
- `N_REPLICATES = 100`
- `N_CUES = 3`
- Fixed params for all replicates (moderate values that produce meaningful behavior):
  - `OMEGA_2 = -3.0`
  - `OMEGA_3 = -6.0`
  - `KAPPA = 1.0`
  - `BETA = 2.0`
  - `ZETA = 0.5`

**Test function: `test_valid04_simulation_equivalence`**
Marked `@pytest.mark.slow`.

1. Load config: `config = load_config()`
2. Generate ONE trial sequence (same env for all replicates):
   `trials = generate_session(config, seed=0)`
   `cue_probs_arr = jnp.array([t.cue_probs for t in trials])`
   
   **Design note (add as a code comment in the test):** Using a single fixed trial
   sequence for all 100 replicates is a deliberate controlled-comparison design.
   This isolates the effect of agent RNG (choice and reward sampling) from environment
   RNG (trial sequence generation). By holding the environment constant, the KS test
   compares only the behavioral difference between the NumPy and JAX agent paths.
   Varying environment seeds across replicates would conflate environment variability
   with implementation differences, reducing the test's sensitivity to detect bias.

3. Extract phase labels for grouping:
   `phase_labels = [t.phase_label for t in trials]`
   `unique_phases = sorted(set(phase_labels))`
   Also compute `phase_indices`: dict mapping phase_label -> list of trial indices.

4. For each replicate seed in `range(N_REPLICATES)`:

   a. **Legacy path** (`simulate_agent`):
      - `net = build_3level_network(omega_2=OMEGA_2, omega_3=OMEGA_3, kappa=KAPPA)`
      - `rng = np.random.default_rng(seed)`
      - `result = simulate_agent(net, trials, beta=BETA, zeta=ZETA, rng=rng)`
      - Extract choices as numpy array

   b. **JAX path** (`simulate_session_jax`):
      - `rng_key = jax.random.PRNGKey(seed)`
      - `choices_jax, _, _ = simulate_session_jax(jnp.float32(OMEGA_2), jnp.float32(OMEGA_3), jnp.float32(KAPPA), jnp.float32(BETA), jnp.float32(ZETA), cue_probs_arr, rng_key)`
      - Convert to numpy

   c. **Compute per-cue per-phase choice frequencies**:
      For each phase_label and each cue (0, 1, 2), compute the fraction of trials
      in that phase where that cue was chosen.
      Append these fractions to per-cue-per-phase accumulator lists
      (separate for legacy and JAX).

5. **KS tests**: For each (phase_label, cue) pair:
   - `stat, p = ks_2samp(legacy_freqs[phase][cue], jax_freqs[phase][cue])`
   - Collect all p-values and all (phase, cue, stat, p) tuples for diagnostics

6. **Assertions**:
   - For each (phase, cue) pair, assert `p > 0.05`:
     ```python
     failures = [(ph, cue, stat, p) for ph, cue, stat, p in results if p <= 0.05]
     assert len(failures) == 0, (
         f"VALID-04 FAILED: {len(failures)} KS tests had p <= 0.05. "
         f"Failures: {failures}"
     )
     ```
   - Also assert that the number of replicates is correct (sanity check):
     `assert len(legacy_freqs[phase][cue]) == N_REPLICATES`

**Additional sanity test: `test_valid04_choice_range`**
Marked `@pytest.mark.slow`. Quick sanity check that both paths produce choices in {0, 1, 2}
for a single replicate. This catches gross implementation bugs before the long 100-replicate
test runs.

- Run both paths with seed=42
- Assert all choices in {0, 1, 2} for both paths
- Assert choice arrays have length == n_trials for both paths

**Note on seed matching**: The NumPy and JAX paths use different RNG backends.
Using the same integer `seed` does NOT produce matching random sequences. The test
does NOT require per-trial match — only that the DISTRIBUTIONS of choice frequencies
are statistically equivalent. Each replicate seed just needs to be consistent within
its own path (deterministic given seed).
